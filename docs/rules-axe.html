ARIA Widgets

Rule ID OpenA11y Rule Summary Related aXe Rules
Widget 1 Accessible name is required
  • Ensure every ARIA button, link and menuitem has an accessible name (Serious)
  • Ensure every ARIA input field has an accessible name (Serious)
  • Ensure every ARIA meter node has an accessible name (Serious)
  • Ensure every ARIA progressbar node has an accessible name (Serious)
  • Ensure every ARIA toggle field has an accessible name (Serious)
  • Ensure every ARIA tooltip node has an accessible name (Serious)
Widget 3 role must be valid
  • Ensure elements do not use deprecated roles (Minor)
  • Ensure all elements with a role attribute use a valid value (Critical)
Widget 4 Values must be valid
  • Ensure ARIA attributes are used as described in the specification of the element's role (Serious)
  • Ensure all ARIA attributes have valid values (Critical)
Widget 5 Attributes that start with aria- must be defined.
  • Ensure an element's role supports its ARIA attributes (Critical)
  • Ensure ARIA attributes are used as described in the specification of the element's role (Serious)
  • Ensure ARIA attributes are not prohibited for an element's role (Serious)
Widget 6 Widgets must set properties
  • Ensure ARIA attributes are used as described in the specification of the element's role (Serious)
  • Ensure elements with ARIA roles have all required ARIA attributes (Critical)
Widget 7 Widgets must have owned elements
  • Ensure elements with an ARIA role that require child roles contain them (Critical)
Widget 8 Role must have parent
  • Ensure elements with an ARIA role that require parent roles are contained by them (Critical)
Widget 12 Accessible name is descriptive
  • Ensure ARIA attributes are used as described in the specification of the element's role (Serious)
Widget 14 Unsupported and deprecated ARIA attributes.
  • Ensure elements do not use deprecated roles (Minor)
